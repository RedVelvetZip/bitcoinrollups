Description

Bitcoin rollup is a zk rollup that provides Zcash-like transactions for Bitcoin users. Users can move BTC in the rollup by way of a trusted, federated multi-sig. Or, they can use the rollups native token ZT, which has a circulating supply of 50,000,000 tokens.

Risk Analysis

State validation

ZK Proofs (ST & SN)

zkSTARKS are zero knowledge proofs that ensure state correctness. zkSNARKS ensure privacy. This rollup uses zkSNARKs to encrypt user data related to transactions, and then verifies that computation within an SNARK to STARK verifier. The STARK then proves state correctness to Bitcoin.

Data availability

On Chain

All of the data regarding the state of the rollup is published on Bitcoin.

Upgradeablilty

Yes.

The system can be changed whenever. Upgrades are managed by a centralized company.

Sequencer failure

Enqueue via L1

Users can force transactions to be included in the Layer 1. The sequencer has a decentralized sequencer, but one that only scales up to 50 nodes. In the event of an attack on the sequencer, users can force inclusion of transactions on the Bitcoin layer 1.

BTC Bridge

Custodial

Users trust a federated multi-sig of 15 signers to custody the Bitcoin used to mint wrapped Bitcoin on the layer 2 protocol. If users don't want to take on this level of trust, they can use the rollups native token ZT.

Rollup Stage

Zeta is a Stage 0 rollup

STAGE 0 has been fully met with the following requirements:

  • The project identifies as a rollup.
  • L2 state roots are published to Bitcoin L1.
  • State transition inputs are available on L1.
  • Source code is available for verification.

STAGE 1 has 1 pending issues:

  • Rollup has decentralized its sequencer.
  • Upgrade consensus mechanism for better security.

STAGE 2 is pending with one major issue:

  • Current state of Bitcoin makes the rollup unable to finalize the integration of a trustless bridge with Bitcoin mainnet.

Technology

Zero-knowledge proofs ensure privacy and state correctness.

Each transaction sees a user generate a ZK-SNARK to ensure their transaction data is encrypted. Each update to the system state must be accompanied by a ZK-STARK proof that ensures that the new state was derived by correctly applying a series of valid user transactions to the previous state. These proofs are then verified by the nodes in the sovereign rollup, and the data is then posted on Bitcoin.

State Derivation

Node software

The node software is open-source, and its source code can be found here. The main node software does not rely on Layer 1 (L1) to reconstruct the state, but you can use this tool for that purpose. Currently, there is no straightforward method to inject the state into the main node, but Zeta is actively working on a solution for this.

Operator

The system has a decentralized sequencer operator

There is a consensus network of nodes that operates the sequencer. 50 nodes build and propose rollup blocks to be executed.
